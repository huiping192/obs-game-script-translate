# Switch 游戏学英语助手 —— 项目构想

## 一句话描述

通过实时分析 Switch 游戏画面，自动识别游戏中的英文对话并给出翻译和词汇解析，让玩家在游戏过程中顺带学英语。

---

## 核心思路

```
Switch 画面采集 → 截图 → AI 分析英文 → 显示结果
```

---

## 技术方案

### 方案一：OBS 插件（目标）

开发一个原生 OBS 插件，直接集成到 OBS 生态中。

**优点**
- 用户体验好，无需额外软件
- 可直接访问 OBS 视频帧，性能好
- 可叠加字幕层到推流画面中

**技术栈**
- 语言：C++，基于 `obsproject/obs-plugintemplate`
- 核心 API：`obs_source_get_frame`、`obs_add_tick_callback`
- AI：Claude Vision API（异步线程调用，避免阻塞主线程）

### 方案二：Python 独立程序（原型验证）

- 指定本地图片作为输入
- 调用 Claude API 分析英文
- 控制台输出结果

---

## AI 选型

**选用 Claude Sonnet**

- 识图能力强，游戏截图的 UI、对话框识别准确
- 价格适中，一张游戏截图约 $0.01 以内
- API 文档友好，调用简单

---

## 原型目标（阶段一）

最小可跑通的流程：

```
指定本地截图文件
    ↓
读取图片 → base64 编码
    ↓
发送给 Claude Vision API
    ↓
输出：原文 / 翻译 / 词汇解析
```

**不包含：**
- 变化检测
- 悬浮窗 UI
- OBS 集成

---


## 主要挑战

| 挑战 | 说明 | 应对思路 |
|------|------|----------|
| API 延迟 | 截图→分析→显示约 1-3 秒 | RPG 类游戏节奏慢，可接受 |
| API 费用 | 频繁调用 Vision API 有成本 | 后期加入变化检测降低频率 |
| 打字动画 | 文字逐字出现，过早截图内容不全 | 等待画面稳定后再截图（后期） |
| 异步处理 | 不能在主线程等待 HTTP 响应 | 独立线程处理 API 调用（OBS 阶段） |

---

## 开发路线

```
阶段一（当前）— Python 原型
  └─ 输入本地图片
  └─ 调用 Claude API
  └─ 控制台输出翻译 + 词汇解析

阶段二 — 接入实时画面
  └─ 用 mss 截取采集卡窗口
  └─ 加入对话框变化检测（画面变化超过比如10%以上）
  └─ 等待画面稳定后再截图

阶段三 — OBS 插件化
  └─ 用 obs-plugintemplate 搭 C++ 工程
  └─ 将 Python 检测逻辑翻译为 C++（像素遍历 + libyuv）
  └─ HTTP 请求用 libcurl（OBS 已内置）
  └─ 结果写入 OBS Text Source 或自定义 overlay
```
